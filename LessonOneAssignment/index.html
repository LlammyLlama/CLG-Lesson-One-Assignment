Lesson One - HTML & CSS

HTML - Hyper Text Markup Language

HTML provides or sets up the base content (text, numbers) that is observed on a web page.

A 'mark up language' is a coding language that uses tags to catergorise or structure the content in a meaningful way.

The 'hyper text' refers to the use of clickable links that lets the user jump from one lot of information to another and describes the type of mark up language this is.

HTML elements

Elements are groups of content categorised by tags

E.g. [opening tag] [content] [closing tag]

Tags

Have angled brackets < > that mark the start ('opening' tag) and angled brackets with a forward slash < / > that mark end ('closing' tag) of an element.

You can have different categories of elements with their associated tags:

Head section with < head > opening tag and < /head > closing tag. Has info like author and title in the address bar - not directly displayed on page.

Body section - < body > < / body > part of the html document that will be displayed.

Heading element with # level < h1 > opening tag and < /h1 > closing tag. This is the header viewed on top of the page. You can have h1 as most important (largest) all the way up to h6 least important which would usually be used as a subheadings

Paragraph element with < p > opening tag and < /p > closing tag. This is the general default text.

Image element < img src="ada.jpg" > does not have a closing tag. It doesn't have content but has attributes. src is "source" attribute

Tip - you can hover over the tag and click MDR properties in Visual Studio Code to get more information about it.

Click this hyperlink: Website for some more common HTML tags.

Attributes

Appear in the opening tag and provides more information to the overall element

Anchor elements are used for hyperlinks. E.g. href attribute can look like:

< a href="www.google.com" > Search < / a >

and tells the browser the address in which the link goes to.

HTML documents begin with a DOCTYPE declaration which tells the browser which version of HTML the document is written - this document uses HTML 5 which has the declaration of < !DOCTYPE html >

html section - < html > and < /html > tells the browser where the document begins and ends

CSS - Cascading Style Sheets

Controls the design aspects like colour, size, styles, fonts, etc. It has rules that are called selectors,properties and values.

Selectors

Which elements does the following rule apply to.

E.g. the 'p' in

p { color:red; }

is the selector and means that all the paragraph HTML elements will be coloured red.

Properties

The property what part of the element is being changed. In the above example, the colour is being changed.

You can have multiple properties separated by a semi colon

E.g.

p { color:red; font-size:12px }

Values

The value is what you're telling the property to change to - in the above example, you want the color property to change it to 'red' and you want the font-size property to be 12px.

Methods of CSS styling

In line styling

Puts the attribute in line with the tag. Messy lines and rarely one style applies to one thing so hard to make changes of the same category/tags. But may be applicable to single document cases ...like emails.

E.g. < h1 style =”color: red;” >my first website < / h1 >

< Style > tag

might end up with 1000 lines of CSS code and not as easily read
